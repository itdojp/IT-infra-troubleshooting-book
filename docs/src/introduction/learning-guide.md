# 学習の進め方

本書は「体系的に学ぶ」用途と、「障害時の参照（リファレンス）」用途の両方を想定しています。

## 1. 体系的に学ぶ（初めて体系化する人向け）

- 第1章: 障害対応の思考法と心構え（共通フレームワーク）
- 第2章: 情報収集（事実の取り方と記録）
- 第3章: 切り分け（仮説の立て方と検証）
- 第4章: レイヤー別の実践（ネットワーク/OS/アプリ/DB）
- 第5章: クラウド/コンテナ特有の観点
- 第6章: 振り返りと再発防止（運用プロセス）

## 2. 参照として使う（運用中の人向け）

1. まず第2章・第3章で「情報をどう集め、どう切り分けるか」を再確認します。
2. 次に、第4章・第5章を「対象レイヤーのチェック観点集」として参照します。
3. 事後対応では第6章を参照し、学びを再発防止へ接続します。

付録は、状況に応じて辞書的に参照してください。

- 付録A: コマンドリファレンス（調査時に「次に何を叩くか」を探す）
- 付録B: チェックリスト（抜け漏れ防止）
- 付録C: 設定ファイル例（設定の観点を思い出す）

## 3. 想定学習時間とチェックリストの扱い

- 各章の「想定学習時間」は、本文の読解だけでなく、章末チェックリストの確認や手元での軽い演習（コマンド例を眺める/試す等）を含む目安です。本文を読むだけなら短くなる場合があります。
- チェックリストは「チェックを付ける」こと自体が目的ではありません。各項目について、可能なら「何を見たか（ログ/コマンド/ダッシュボード）」と「結論（正常/異常/保留）」を1行で残すと、引き継ぎや振り返りで再利用しやすくなります。

## 4. 本書の入口

本書の入口は `docs/index.md`（目次）と `README.md`（概要）です。まずは全体像を把握し、目的に応じて読み進めてください。

# 本書の目的

本書『ITインフラトラブルシューティング大全』は、システム障害に直面したときに「何から確認し、どう切り分け、どの順で手を打つか」を整理できるようになることを目的とした実践的ガイドです。

本書では、単にコマンドを列挙するのではなく、以下を重視します。

- 症状から原因候補を立て、検証して絞り込むための思考プロセス
- ネットワーク/OS/ミドルウェア/アプリケーション/DB/クラウドといったレイヤー別の確認観点
- インシデント対応後に学びを蓄積し、再発防止につなげるための運用プロセス

## 対象読者（要約）

本書は、Linux やネットワークの基礎を一通り学んだ「入門修了レベル」以降の読者を主な対象としています（詳細は `docs/index.md` の「対象読者」を参照してください）。
